        넣었다 지웠다하는 메모리
 
        참조 타입 변수를 집어넣음
 
        참조 타입 변수:
            기본타입 변수를 제외한 나머지
 
            대표적으로 String 참조타입
 
                String은 무조건 하나의 null(빈공간)이 포함되어있다.
                String "A"이면 [A.null]
                String "홍길동"이면 [홍.길.동.null] 이런식으로
 
                String name1 "홍길동" 하면 
                Stack 에 홍길동이란 이름이 생기고
                Stack 안에 name1값에
                            name1[100] //100은 예시
                            이란 값을 가지는데
                100은 Heap 안에 100이란 번지 안에 [홍,길,동,null]이란 값이 담겨있다.
 
                만약     이상황에 name2를 선언 하면 
                Stack 안에 name2[200]
                Heap 200이란 번지를 만들어 두는데
                name2에 똑같이 "홍길동" 넣어주면
 
                name2[]에 []엔 Heap 안에 name1이 쓰고있는 100 안에 [홍,길,동,null]의 주소값이 나타난다.
 
                name1 과 name2가 같냐고 하면 자바에서는 같다고 하지만 주소값이 같아서 같다하는것이지
 
                "홍길동" 이란 글자가 같아서 같다고 하는것이 아니다.
 
                이떄 new String();로 Heap 영역에 다른 객체를 만들어 주면 Heap 안에 주소값이 달라진다.
 
                예시로
 
                String name3 = new String("홍길동");
 
                을하면 name1 name3 둘다 홍길동이란 글자이지만 Heap 안에 주소의 값은 다르므로 자바는 둘이 다르다 라고인식한다.
 
            하지만 if(str1.equals(str3)) 처럼 equals문으로 비교를 하면 Heap 안에 주소안에 값인 "홍길동" 을 비교하므로
            같다 라고 뜬다.
 
 
        
 
        NullPointerException
 
            예외(Exception)
            사용자의 잘못된 조작 이나 잘못된 코딩으로 인해 발생하는 프로그램 오류
 
            NullPointerException
            참조 변수가 null 값을 가지고 있을 때
            객체의 필드나 메소드를 사용하려고 했을 때 발생      //jsp에서 정말 많이 발생
            
            값을 주지 않고 처리했을떄 일어난 문제
